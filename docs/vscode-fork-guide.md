# VS Code Fork Research Guide

Complete guide for forking VS Code as the base for PocketShell Desktop.
Generated by research agent from live upstream data.

## Summary

- **Repo**: https://github.com/microsoft/vscode (MIT license, ~1.1 GB)
- **Node.js**: 24.15.0 (pinned in `.nvmrc` — the wiki says >=22.x but the actual pinned version is 24.15.0)
- **Build**: `npm install && npm run watch` (dev), `npm run gulp vscode-{platform}-{arch}` (prod)
- **Branding**: `product.json` at repo root + icon assets in `resources/`
- **Patch strategy**: Follow VSCodium's approach — git patches applied after upstream sync
- **Marketplace**: Use OpenVSX (official marketplace blocks forks)

## Key Files to Modify

| File | Purpose |
|------|---------|
| `product.json` | App name, data folder, URLs, built-in extensions |
| `resources/` | Application icons (Linux, Windows, macOS) |
| `src/vs/workbench/browser/actions/developerActions.ts` | About dialog branding |
| `src/vs/platform/product/common/product.ts` | Product identity |

## VSCodium Patches (Reference)

- `brand.patch` (141 KB) — main rebranding across source
- `cli.patch` — CLI binary name
- `binary-name.patch` — output binary names
- `telemetry.patch` — disable telemetry
- `fix-gallery.patch` — point to OpenVSX
- `disable-copilot.patch` — remove Copilot
- `disable-cloud.patch` — remove cloud features

## Concrete Steps

1. Fork microsoft/vscode on GitHub
2. `git clone --depth=1` the fork
3. Modify `product.json` (nameShort, nameLong, applicationName, dataFolderName, etc.)
4. Replace icons in `resources/`
5. Search/replace "Code - OSS" and "Visual Studio Code" strings
6. `npm install && npm run watch`
7. Test with `./scripts/code.sh`
8. Production build: `npm run gulp vscode-linux-x64-min`

## Trade-offs

- **Maintenance**: VS Code releases monthly; patches must be updated
- **Extensions**: Proprietary extensions (Remote Dev, Live Share) may not work
- **Alternative**: Eclipse Theia (VS Code extension compat, designed as platform)
